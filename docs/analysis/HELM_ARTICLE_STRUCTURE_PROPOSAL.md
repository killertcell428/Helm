```markdown

## はじめに

Helmは、**組織構造そのものを学習・改善するAI**です。
既存のAIエージェントがタスクや成果物を最適化するのに対し、
Helmは**組織構造そのものの壊れ方**を見て、改善します。

本記事では、Helmの設計思想と技術的実装について解説します。

## 1. 背景・課題：組織の意思決定における構造的盲点

### 1.1 現状の課題（定量データで提示）

- **意思決定リードタイム**: 7-14日（大企業の典型的な意思決定プロセス）
- **Zombie Project検知率**: 0%（構造的問題が検知されない）
- **代替案検討率**: 10%未満（初期仮説への固執）
- **経営層レビュー負荷**: 15h/週（手作業での監視）

### 1.2 3つの構造的課題

#### 課題1: 責任所在の不可視化
大企業では、「この仮説は棄却すべきか」「この事業は撤退すべきか」
という痛みを伴う判断の責任者が曖昧になります。

#### 課題2: 構造固定化バイアス
初期仮説への固執が組織的なバイアスとなり、
実験データや市場反応が乏しくなっても、構造的な軌道修正が起きません。

#### 課題3: ヒト主語の認知限界
実験ログ・会議ログ・KPI変更履歴など、膨大なシグナルを
人が手作業で監視する必要があります。

### 1.3 既存ソリューションの限界

- **Asana、Jira**: タスクや成果物を最適化するが、組織構造は見ない
- **CrewAI、AutoGen**: マルチエージェントでタスクを実行するが、
  組織の意思決定プロセスそのものは改善しない
- **BIツール**: データを可視化するが、構造的問題は検知しない

## 2. Helmとは：AIが人を呼び出す逆転の発想

### 2.1 コンセプト

**「AIや人ではなく組織を賢くする」**

- **既存ツール**: タスクや成果物を最適化
- **Helm**: 組織構造（判断構造・責任境界・役割分担）を観測・評価・再設計

### 2.2 差別化ポイント

1. **AIが人を呼び出す**（従来型の逆転）
   - 構造的問題を検知すると、組織グラフと責任モデルに基づいて
     自動的に適切なロール（Executive等）を呼び出す

2. **マルチ視点評価システム**
   - 4つのロール視点（Executive, Corp Planning, Staff, Governance）から
     同一データを評価し、アンサンブルスコアリングで統合

3. **定量評価による説明可能なアラート**
   - テキストから評価基準を自動抽出し、スコア（0-100点）に基づいて評価
   - アラート理由を分かりやすく説明

4. **メタ評価としての統合判断**
   - 複数情報源（Google Meet議事録、Google Chat、スライド、データ分析）を
     統合的に評価し、「部分的な最適解」ではなく「組織全体を見た判断」を提供

### 2.3 Human in the Loopの設計思想

Helmは完全自動化を目指すのではなく、**Human in the Loop**の設計思想を持ちます：

- **検知**: Helmが構造的問題を検知（スコア70点以上）
- **エスカレーション**: 適切なロール（Executive等）に自動通知
- **判断**: 人間（Executive）が介入案を承認/却下
- **実行**: 承認後、AIが自律的にタスクを実行（市場データ分析、資料生成等）

この設計により、**「人は舵を取る。AIは船でありパドルでありレーダーである」**
という新しいヒト×AIの共生関係を実現します。

## 3. 機能と実装

### 3.1 構造的問題検知システム

#### 検知パターン

1. **B1_正当化フェーズ**
   - KPI下方修正が2回以上
   - 撤退議論なし
   - 判断集中率40%以上

2. **ES1_報告遅延**
   - リスク提起メッセージ存在
   - エスカレーション未完了
   - 判断集中率50%未満

3. **A2_撤退判断の遅れ**
   - 事業悪化が明らかなのに撤退・ピボット議論が行われない
   - （主にLLM検出）

#### マルチ視点評価システム

```
[構造化データ]
      │
      ├─────────────────────────────────────┐
      │                                     │
      ▼                                     ▼
[ルールベース分析]              [マルチ視点LLM分析]
[StructureAnalyzer]              [MultiRoleLLMAnalyzer]
      │                                     │
      │                                     ├─→ [Executive視点] (重み: 0.4)
      │                                     ├─→ [Corp Planning視点] (重み: 0.3)
      │                                     ├─→ [Staff視点] (重み: 0.2)
      │                                     └─→ [Governance視点] (重み: 0.1)
      │                                     │
      │                                     ▼
      │                              [各ロールの評価結果]
      │
      └─────────────────────────────────────┘
                      │
                      ▼
            [アンサンブルスコアリング]
            [EnsembleScoringService]
                      │
                      ├─→ スコア計算: 0.6 × ルールベース + 0.4 × LLM平均
                      ├─→ 重要度・緊急度: 安全側（最も強い）を採用
                      └─→ 説明文: ルールベース + 主要ロールのコメント
                      │
                      ▼
            [パターン検出結果]
```

### 3.2 Executive呼び出しフロー

```
[アラート生成]
      │
      ▼
[エスカレーション判断]
      │
      ├─→ [責任モデル参照]
      ├─→ [組織グラフ参照]
      └─→ [ロール選択]

[Executive呼び出し]
      │

      │
      ├─→ [承認] ──→ [AI実行開始]
      └─→ [却下] ──→ [終了]
```

### 3.3 AI自律実行（ADKベースのマルチエージェントシステム）

Executiveの承認後、Helmは**ADK (Agent Development Kit)** を使用した
マルチエージェントシステムで自律的に以下のタスクを実行します：

- **市場データ分析** (ResearchAgent): 市場データを収集・分析
- **社内データ統合** (AnalysisAgent): 社内データを統合し、財務シミュレーションを実行
- **資料生成** (DocumentAgent): 3案比較資料を自動生成
- **通知送信** (NotificationAgent): 関係部署への通知を送信
```
