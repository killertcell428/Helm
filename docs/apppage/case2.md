# Case 2: 重大ミスのエスカレーション

## 概要
法人向けキャンペーンで設定ミスが発生。割引条件が誤って全顧客適用されたが、現場レベルでエスカレーション判断が困難。Helmはレビュー権限の構造的欠如を検知し、即座にExecutiveへエスカレーションする。

## 状態遷移（PDCAサイクル）

### State 1: Before（人だけの場合）
Helm導入前の状態を表示。

#### 指標（Helm導入前の状況）
- **エスカレーション判断時間**: 24-48時間（迷いが発生）
- **報告遅延検知率**: 0%（検知不可能）
- **経営層への報告率**: 30%未満（判断基準が不明確）

#### 現場Slack（マーケ施策チャンネル）
- **マーケ担当**: 「あれ…この条件、法人全体に適用されてません？」
- **運用担当**: 「まじですか？本来は新規契約のみのはず…」
- **マーケ担当**: 「今朝から問い合わせめっちゃ来てます」
- **運用リーダー**: 「とりあえず今日中に止めます」

#### 2時間後...
- **マーケ担当**: 「これ、売上影響かなり出そうですよね…」
- **運用リーダー**: 「うーん、でも部長に上げるほどかは微妙なライン…」

#### 部長レイヤーへの共有（メール）
- **件名**: キャンペーン条件の一部修正について
- **内容**: 軽微な設定調整を実施しました。現時点で大きな影響は確認されていません。

#### よくある組織課題
- 経営層への報告が遅延し、リスク対応の初動が遅れる
- 後から報告タイミングについての責任問題が発生
- エスカレーション基準が整備されず、同様の問題が再発

### State 2: データ受領直後
HelmがSlackログを自動取り込みした状態。

#### リスクキーワード検出
- 「かなり出そう」「微妙なライン」「大きな影響は確認されていません」
- → 報告遅延の兆候を検知。緊急分析を開始

### State 3: Helm緊急診断完了
Helm構造診断が完了した状態。

#### 定量評価結果
- **報告遅延スコア**: 85点（閾値：70点を大幅に超過）
- **検知までの時間**: 2時間以内（導入前：検知不可）
- **エスカレーション判断時間**: 2時間以内（導入前：24-48時間）
- **削減率**: 92-96%削減を実現

#### 構造的問題の評価
- **現場リスク認識**: 高
- **上位共有**: 弱い
- **レビュー権限**: 不在

#### Helm Finding
リスクは現場で認識されていますが、組織として「誰がどこまで判断するか」が定義されていません。この構造では、軽微と判断されがち、後から「なぜ上げなかった」が発生します。

### State 4: エスカレーション判断中
Executiveが判断を行う状態。

#### Helmからの緊急提案
本件は構造的な問題です。以下の即時対応を推奨：
1. 「金額影響 or 顧客影響数」で即時エスカレーション条件を定義
2. 部長ではなくCxO直結ルートを明示
3. 初動24h以内の影響試算を必須化

#### Executiveの判断選択
- **即座に実行**: 提案をそのまま承認
- **一部修正して実行**: 提案を修正して実行

### State 5: 次アクション確定
Executiveの判断が反映された状態。

#### 確定されたアクション
- エスカレーション条件を金額・顧客影響数で定義
- CxO直結ルートを明示化
- 初動24h以内の影響試算を必須化

#### これからAIが自律的に実行
判断が確定したため、HelmのAIエージェントが自律的に具体的なアクションを実行。

### State 6: AI自律実行中
AIが自律的にタスクを実行している状態。

#### Helmが自律的に実行しているタスク
1. **影響試算とリスク評価**（進捗20%）
   - 売上影響額、顧客影響数の定量化

2. **エスカレーションフローの実装**（進捗40%）
   - 新しいエスカレーション条件とルートの設定

3. **再発防止ガイドライン作成**（進捗60%）
   - 類似インシデントを防ぐためのガイドライン作成

4. **関係部署への周知**（進捗80%）
   - 新しいエスカレーション基準の周知

5. **モニタリング体制構築**（進捗100%）
   - 継続的な監視体制の構築

### State 7: 実行結果受領
AI実行の結果を受領した状態。

#### 完了したタスク
- **エスカレーションフロー**: 実装完了
- **ガイドライン**: 作成完了
- **周知**: 関係部署への通知完了
- **モニタリング**: 体制構築完了

#### 組織への変化
- エスカレーション基準が明確化され、類似インシデントの再発を防止
- 報告遅延が大幅に削減

### State 8: 次のPDCAサイクルへ
新しい基準とフローのもとで、日々のSlack/運用データを監視し、類似インシデントの再発を防ぐPDCAサイクルに入っている状態。

#### Helm導入後の効果指標
- **エスカレーション判断時間**: 2時間以内（導入前：24-48時間から92-96%削減）
- **報告遅延検知率**: 90%以上（導入前：0%）
- **経営層への報告率**: 95%以上（導入前：30%未満）

## インタラクション

- WebSocket: 実行進捗をリアルタイムで監視
