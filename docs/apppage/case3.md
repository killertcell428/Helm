# Case 3: 外部トレンドのチャンス検知

## 概要
競合C社が自治体向けAI通信基盤を発表。社内では機会として認識されるも、資源再配分の決定権限が不明確。Helmは決定者不在による機会損失リスクを検知し、迅速なピボット提案を行う。

## ページ構成

### 1. ナビゲーションバー
- **Helmロゴ**: CompassアイコンとHelmブランド名
- **戻るボタン**: 「デモ一覧に戻る」ボタン（/demoへリンク）

### 2. ヘッダー
- **バッジ**: 「Case 3」
- **タイトル**: 「外部トレンド：機会認識と決定権限の不一致」
- **説明**: 競合C社が自治体向けAI通信基盤を発表。社内では機会として認識されるも、資源再配分の決定権限が不明確。Helmは決定者不在による機会損失リスクを検知します。
- **現在の状態バッジ**: 現在のPDCAサイクルの状態を表示

### 3. AIの裏側サマリ
- **説明**: 現在の状態に応じて、HelmのAIが何をしているかを表示

## 状態遷移（PDCAサイクル）

### State 1: Before（人だけの場合）
Helm導入前の状態を表示。

#### 指標（Helm導入前の状況）
- **意思決定リードタイム**: 3-6ヶ月（検討が長期化）
- **機会認識と権限不一致検知率**: 0%（検知不可能）
- **資源再配分の実現率**: 20%未満（決定権限が不在）

#### 外部ニュース
- **タイトル**: 競合C社、自治体向けAI通信基盤を発表
- **内容**: スマートシティ案件を複数受注

#### 社内Slack（横断チャンネル）
- **新規事業**: 「これ、うちのIoT基盤と相性いいですよね」
- **DX本部**: 「PoCならすぐ行けそう」
- **通信本部**: 「ただ今期は設備投資かなり厳しいですよ」

#### 企画会議メモ
- 面白いテーマ
- ただ既存施策との優先度整理が必要
- 誰が最終判断するかは次回検討

#### よくある組織課題
- 意思決定プロセスが長期化し、市場機会を逃す
- 既存施策の優先度調整ができず、リソース配分が硬直化
- 競合他社に先行され、後追いのポジションに

### State 2: データ受領直後
HelmがニュースフィードとSlackを自動取り込みした状態。

#### チャンスシグナル検出
- 「相性いい」「すぐ行けそう」vs「次回検討」
- → 機会認識と決定権限の不一致を検知。緊急分析を開始

### State 3: Helm機会分析完了
Helm構造診断が完了した状態。

#### 構造的問題の評価
- **チャンス認識**: 高
- **資源再配分決定者**: 不在
- **決定期限**: なし

#### Helm Finding
機会は認識されていますが、計画と権限の構造が一致していません。このままでは検討が長期化し、競合に先行される可能性が高いです。

#### 定量評価結果
- **機会認識と権限不一致スコア**: 80点（閾値：70点を超過）
- **検知までの時間**: 24時間以内（導入前：検知不可）
- **意思決定速度**: 2日以内（導入前：3-6ヶ月）
- **削減率**: 98-99%削減を実現

### State 4: Executiveの判断
Executiveが判断を行う状態。

#### Helmからの提案
機会損失を防ぐため、以下の即時対応を推奨：
1. **資源再配分権限の明確化**: Executive直下の意思決定チームを設置
2. **決定期限の設定**: 30日以内にGo/No-Go判断
3. **PoC計画の具体化**: 2週間以内にPoC開始

#### Executiveの判断選択
- **即座に実行**: 提案をそのまま承認
- **一部修正して実行**: 提案を修正して実行

### State 5: 次アクション確定
Executiveの判断が反映された状態。

#### 確定されたアクション
- 資源再配分権限をExecutive直下の意思決定チームに明確化
- 30日以内にGo/No-Go判断の期限を設定
- PoC計画を具体化（2週間以内に開始）

#### これからAIが自律的に実行
判断が確定したため、HelmのAIエージェントが自律的に具体的なアクションを実行。

### State 6: AI自律実行中
AIが自律的にタスクを実行している状態。

#### Helmが自律的に実行しているタスク
1. **市場機会分析**（進捗20%）
   - 競合動向、市場規模、参入障壁の分析

2. **リソース最適化案**（進捗40%）
   - 既存施策との優先度調整、リソース再配分案の作成

3. **関係部署調整**（進捗60%）
   - 新規事業、DX本部、通信本部との調整

4. **PoC計画とKPI設定**（進捗80%）
   - PoCの具体的な計画とKPI設定

5. **決定期限のカレンダー登録**（進捗100%）
   - 30日後のGo/No-Go判断会議の設定

### State 7: 実行結果受領
AI実行の結果を受領した状態。

#### 完了したタスク
- **市場機会分析**: 完了
- **リソース最適化案**: 作成完了
- **関係部署調整**: 調整完了
- **PoC計画**: 作成完了
- **カレンダー登録**: 完了

#### 組織への変化
- 機会認識から実行判断までのプロセスが明確化
- 意思決定速度が大幅に向上（3-6ヶ月から2日以内）

### State 8: 次のPDCAサイクルへ
30日後のGo/No-Go判断に向けてPoCデータを収集しつつ、今後も市場ニュースと社内ログを監視して次の機会シグナルを検知するフェーズ。

#### Helm導入後の効果指標
- **意思決定リードタイム**: 2日以内（導入前：3-6ヶ月から98-99%削減）
- **機会認識と権限不一致検知率**: 90%以上（導入前：0%）
- **資源再配分の実現率**: 80%以上（導入前：20%未満）

## インタラクション
- 状態遷移: 「次へ」ボタンで各状態を順次遷移
- Executive判断: 「即座に実行」「一部修正して実行」を選択
- 戻る: ナビゲーションバーからデモ一覧へ戻る

## 技術仕様
- 状態管理: ローカル状態管理
- WebSocket: 実行進捗をリアルタイムで監視
- フォールバック: WebSocket接続失敗時はポーリング方式に切り替え
