### ① 最小シグナルの組み合わせについて

**現状の設計**：
現在は個別パターン（B1、R1、ES1など）を検知し、メタ評価層で統合的に評価する設計になっています。

**改善計画**：
以下の3つの組み合わせパターンを定義し、実装予定です：

1. **正当化フェーズの兆候**
   - 条件：KPI下方修正2回以上 + 反対意見無視1回以上 + 判断集中度0.7以上
   - 1か月版で実装予定（SHOULD）

2. **エスカレーション遅延の兆候**
   - 条件：報告遅延 + 判断分散 + 反対意見無視2回以上
   - 3か月版で実装予定（MUST）

3. **Zombie Project化の兆候**
   - 条件：KPI下方修正3回以上 + 正当化フェーズ + 判断集中度0.8以上
   - 3か月版で実装予定（MUST）

各メトリクスに重み付けを導入し、組み合わせスコアが閾値を超えた場合にアラートを発火する仕組みにします。

### ② 時系列評価について

**ご理解の確認**：
✅ 「単発ログではなく意思決定の時系列を見ている」という理解で正しいです！

**現状の設計**：
現在は時間範囲指定によるログ集計レベルです。

**改善計画**：
- **1か月版**：基本的な時系列メトリクス（KPI下方修正のトレンド）を1つ実装予定（SHOULD）
- **3か月版**：時系列パターン検出エンジンを実装予定（MUST）
  - 移動平均によるトレンド検出
  - 変化点検出（例：KPI下方修正が急増した時点）
  - 周期性検出（例：四半期ごとのKPI下方修正）

過去4週間などの期間での意思決定パターンを分析し、「判断が遅れ続ける」「KPI修正が連続する」といった時系列パターンを検出します。

**1か月版（中間ハッカソン版）**：
- 時系列評価：時間範囲指定によるログ集計 + 基本的な時系列メトリクス1つ（SHOULD）
- 最小シグナル：個別パターンの検知 + 基本的な組み合わせパターン1つ（SHOULD）

**3か月版（本番大会版）**：
- 時系列評価：時系列パターン検出エンジン（MUST）+ 3つの時系列メトリクス（MUST）
- 最小シグナル：3つの組み合わせパターン（MUST）+ 外部設定ファイル化（SHOULD）

デモでは、1か月版では「時間範囲指定によるログ集計」と「基本的な組み合わせパターン」を見せ、3か月版では「時系列パターン検出」と「複数メトリクスの組み合わせ判定」を見せる予定です。
