## 2. 問題提起：既存組織の意思決定は「構造的に壊れている」（1分）

### 台本

「まず、**既存組織の意思決定は「構造的に壊れている」**という問題から説明します。

組織の意思決定プロセスには、以下の**構造・組織的な問題**が存在します。

**第一の壁：責任境界の曖昧性**

大企業では、研究で『この仮説は棄却すべきか』、ビジネスで『この事業は撤退すべきか』という重要な判断の**責任者が不明確**になります。

**第二の壁：バイアスによる修正困難化**

**忖度・楽観・既得権益による歪み**が組織的なバイアスとなり、実験データや市場反応が乏しくなっても、構造的な軌道修正が起きません。

**第三の壁：人主導によるAI活用の限界**

実験ログ・会議ログ・KPI変更履歴など、膨大なシグナルを人が手作業で監視する必要があります。AIは「呼び出されるだけ」の存在であり、組織の構造的問題を自ら検知することができません。

**調査事例（大企業）**

従業員1,000名以上の企業では、以下の実データが報告されています：

- **平均意思決定リードタイム：70〜85日**
- **新規事業の失敗率：約70%**
- **年間機会損失：約20億円**

これらは、組織構造の問題であり、個々のタスクの問題ではありません。」

### 画面表示

- 数値データを強調表示
- 図解（組織構造の問題を示す）

## 3. どう解決するか：Helmの機能コンセプト（1分）

### 台本

「**Helmは、組織そのものを評価・判断するシステム**を構築します。

従来の
- ❌「人がAIを呼び出す」
- ⬇
- ⭕「AIが人を呼び出す」

へ転換します。

**Helmを構成する3つの中核機能：**

**① 多角的な評価・判断システム**

- **ルールベース（0.6） × LLM（0.4）** のハイブリッド評価
- 以下4視点で同時評価
  - 経営者視点（Executive）
  - 経営企画視点（Planning）
  - 現場視点（Staff）
  - ガバナンス視点（Governance）
- 見落としのない**統合判断**を実現

**② AIが人を呼び出し、判断を依頼**

- 構造的問題を検知すると、AIが自動で**役員 / 部長 / スタッフ**を特定
- 判断・承認を依頼
- 人がAIを監視するのではなく、**AIが組織を監視し、人に判断を求める**

**③ 学習・改善PDCAの自動化**

以下のループを自動で回します：

```
観測 → 評価 → 介入 → 実行 → 再観測
```

- 組織構造の「壊れ方」を学習
- 時間とともに**組織自体が賢くなる**」

### 画面表示

- Helmのアプローチを図解
- 数値データを強調表示
- 「AIが人を呼び出す」の概念図

### 4.3 構造的問題検知（1分30秒）

### 台本

「**Helm解析結果を見る**ボタンをクリックします。

Helmが**マルチ視点評価システム**により構造的問題を検知しました。

Helmは、ルールベース分析と4つの異なる視点から同一のデータを評価します：

1. **経営者視点（Executive）**: 全社の業績・リスク・ステークホルダー責任の観点から評価
2. **経営企画視点（Corp Planning）**: KPI・事業ポートフォリオ・撤退/投資判断の観点から評価
3. **現場視点（Staff）**: 実行可能性と現場負荷の観点から評価
4. **ガバナンス視点（Governance）**: 報告遅延・隠れたリスク・コンプライアンスの観点から評価

これらの評価結果をアンサンブル（統合）することで、人間の経営判断に近い精度で構造リスクを評価します。

検知された問題は、**B1_正当化フェーズ**（KPI悪化が続いているにも関わらず、戦略変更の議論が行われない状態）です。

- KPI下方修正が3回検出
- 撤退/ピボット議論が一度も行われていない
- 判断集中率（最も多く発言した人の発言数 / 総発言数）: 75.00%

**アンサンブルスコアは69点**で、閾値70点に近いため、エスカレーションが必要です。

重大度は**HIGH**、緊急度は**HIGH**です。

画面には、各ロール視点の評価結果も表示されています。例えば、経営企画視点では緊急度が**HIGH**と評価されており、これが最終的な緊急度決定に影響しています。

説明文には、ルールベースの説明と主要ロールのコメントが統合されて記載されています。」

## 5. 技術的なポイント：Hybrid評価アーキテクチャ（1分）

### 台本

「**なぜルール×LLMなのか**

**ルールベースの限界**
- 数値は正常だが「空気が異常」を見逃す

**LLM単体の限界**
- 楽観バイアス・ハルシネーション

**INPUT（ルールベース）**
- KPI下方修正回数 > 2
- 撤退議論キーワード：なし
- 判断集中率（最も多く発言した人の発言数 / 総発言数） > 40%

→ **見逃しゼロの安全網**

**CONTEXT ANALYSIS（LLM）**
- 経営者視点（0.4）：全社リスク
- 経営企画視点（0.3）：ポートフォリオ整合
- 現場視点（0.2）：現場負荷
- ガバナンス視点（0.1）：コンプラ・報告遅延

→ **文脈理解で過剰反応を抑制**

**FINAL DECISION**

```
0.6 × Rule + 0.4 × LLM
Weighted Ensemble Score
```

- 重要度・緊急度は**最も安全側の評価を強制採用**

このアプローチにより、単一の評価軸では見落としがちな問題も、複数の視点から検知できるようになります。**

## 6. 効果と展望：Before/After比較（30秒）

### 台本

「**BEFORE：人手による監視・判断**

- **見落とし**：膨大な会議・チャットログから予兆検知不可
- **遅延**：構造的問題に気づくのが遅い（1〜2週間）
- **バイアス**：忖度・「始めたからには」心理

**AFTER：Helmによる自律支援**

- **監視**：マルチ視点AIが常時ログ監視
- **検知**：構造的な「壊れ方」を即時検知・エスカレーション
- **客観性**：定量スコア × 他視点評価でバイアス排除

**定量的効果（フェルミ推定）：**
- 意思決定リードタイム：**70-85日 → 2日以内（97%削減、35倍改善）**
- Zombie Project検知率：**0% → 90%以上**（Zombie Projectとは、失敗しているにも関わらず継続されているプロジェクトのこと）
- 代替案検討率：**10%未満 → 50%以上（5倍以上改善）**
- 経営層レビュー負荷：**15h/週 → 3h/週（80%削減）**

**将来展望：**
- Phase1：新規事業・R&D（現在）
- Phase2：危機管理・予算配分（6–12ヶ月）
- Phase3：組織変革・M&A（12–24ヶ月）
- Phase4：人事評価・昇進（24–36ヶ月）」

- 定量的効果を数値で強調表示
- 将来展望を図解
