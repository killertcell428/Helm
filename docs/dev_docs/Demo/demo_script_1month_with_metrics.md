## Helm 1か月MVP デモ台本（Biz 新規事業ユースケース）- 定量的成果版

### 0. 導入（口頭）

- 「Helm は、タスクや成果物ではなく、**プロジェクトの進め方そのものが壊れていないか**を見るツールです。」
- 「今日は、健全な新規事業チームと、Zombie Project化しつつあるチームを並べて、Helm がどこを見ているかをお見せします。」
- 「**Helm導入により、意思決定速度が35倍改善し、Zombie Project検知率が0%から90%以上に向上**することを実データに基づく数値で示します。」

### 1. 画面1：ホーム / NWC全体ダッシュボード（`/`）

ナレーション例：

- 「ここがHelmのホーム画面です。舞台は、通信・DX・新規事業を抱える架空企業 NeoWave Communications（NWC）です。」
- 「上のヒーローと3つのカードで、Helmが解決したい **3つの構造的盲点（責任境界の不可視化 / 構造固定化バイアス / 人主語の認知限界）** を一気に示しています。」
- 「**画面右上に、NWC全体の効果指標を表示しています（実データベースのフェルミ推定）：**
  - 意思決定リードタイム：**70-85日 → 2日以内（97%削減、35倍改善）**
  - Zombie Project検知率：**0% → 90%以上**
  - 経営層レビュー負荷：**15h/週 → 3h/週（80%削減）**」
- 「少し下にスクロールすると、NWC内のプロジェクト一覧（Healthy / Zombie）と、3つの業務ケースデモ（経営会議後 / 重大ミス / 外部トレンド）が並んでいます。」

操作：

- ブラウザで `/` を開き、ヒーロー＋3つの構造盲点カードを軽く見せる。
- **効果指標カードを強調して説明**（数値を見せながら）
- その後、画面中央付近の「NWC内プロジェクトの構造ヘルスチェック」セクションで、`Healthy Project (Biz)` と `Zombie Project (Biz)` の2つが見えている状態を見せる。
- `Zombie Project (Biz)` の「構造アラートを見る」ボタンをクリックして `/projects/project_zombie/alerts` へ進む。

### 2. 画面2：Zombie の構造診断一覧（`/projects/project_zombie/alerts`）

ナレーション例：

- 「Zombie Project の構造診断画面です。」
- 「上の段は、Helm が1日の終わりに集計した **構造指標** です。」
  - 「判断の集中度が **70%** を超えていて、ほとんどPMだけが決めています。」
  - 「KPI目標の下方修正が**3回連続**で続いていて、『続ける理由』だけが増えている状態です。」
- 「**画面右上に、このプロジェクトの効果指標を表示しています（実データベースのフェルミ推定）：**
  - Helm導入前：意思決定リードタイム **70-85日**、Zombie Project検知率 **0%**
  - Helm導入後：意思決定リードタイム **2日以内（97%削減、35倍改善）**、Zombie Project検知率 **90%以上**」
- 「下の段が、Helm が付けた **構造的なラベル** です。」
  - 「ここでは『判断の先送り・判断集中』『正当化フェーズ』という2つが検知されています。」

操作：

- 指標3つ（判断集中度/KPI下方修正/反対意見無視）を軽く指差し説明。
- **効果指標カードを強調**（Before/Afterの数値を対比）
- Findings の `B1: 正当化フェーズ（Zombie Project化の兆候）` をクリックして詳細へ。

### 3. 画面3：正当化フェーズの詳細（`/alerts/project_zombie/B1_justification_phase`）

ナレーション例：

- 「ここでは、Helm が『正当化フェーズ』とラベルを付けた理由を、人間が読める文章で説明しています。」
- 「ポイントは、**何が壊れているかを、KPIや決定ログから構造として説明している**ところです。」
- 「単に『売上が悪い』ではなく、KPI目標を下げ続けて、撤退ラインが一度も議論されていない、という構造の問題を指摘しています。」
- 「**画面右側に、定量評価結果を表示しています（実データベースのフェルミ推定）：**
  - 正当化フェーズスコア：**75点**（閾値：70点を超過）
  - 検知までの時間：**Helm導入前：検知不可 → Helm導入後：48時間以内**
  - 意思決定速度：**Helm導入前：70-85日 → Helm導入後：2日以内（97%削減、35倍改善）**」

操作：

- 説明テキストの重要な1〜2行を読み上げる。
- 下の「この時点での構造指標」も合わせて示し、構造と数字が対応していることを伝える。
- **定量評価結果カードを強調**（スコア75点と閾値70点の比較、効果数値を指差し）

次に：

- 「では、この構造が1日の中でどう積み上がっているのかを見てみましょう。」と言って、`/projects/project_zombie/day` に戻る。

## デモでの数値提示のポイント

### 1. 効果指標の表示場所

- **ホーム画面**：NWC全体の効果指標を右上に表示
- **プロジェクト詳細画面**：各プロジェクトの効果指標を右上に表示
- **アラート詳細画面**：定量評価結果と効果数値を右側に表示
- **タイムライン画面**：Before/After比較を下部に表示

### 2. 数値の強調方法

- **カード形式**：数値をカードで囲み、視覚的に強調
- **Before/After対比**：左右または上下に並べて対比
- **パーセンテージ表示**：削減率や改善率を明確に表示
- **グラフ表示**（時間があれば）：棒グラフや折れ線グラフで可視化

### 3. 数値の信頼性

- **シミュレーションデータ**：モックデータに基づく数値であることを明記
- **根拠の説明**：各数値の算出根拠を簡潔に説明
- **実証実験への言及**：3か月版で実証実験を実施予定であることを言及
